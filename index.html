Data Classification

What is Data Classification?

Data classification is the process of organizing data by agreed-on categories. Thoroughly planned classification enables more efficient use and protection of critical data across the organization and contributes to risk management, legal discovery and compliance processes.
Data classification tags data according to its type, sensitivity, and value to the organization if altered, stolen, or destroyed. It helps an organization understand the value of its data, determine whether the data is at risk, and implement controls to mitigate risks. Data classification also helps an organization comply with relevant industry-specific regulatory mandates such as SOX, HIPAA, PCI DSS, and GDPR.



Types of Data Classification

Data is classified according to its sensitivity level—high, medium, or low.

 • High Sensitivity Data: Data that if compromised or destroyed in an unauthorized transaction, would have a catastrophic impact on the organization or individuals.

 • Medium Sensitivity Data: Data intended for internal use only, but if compromised or destroyed, would not have a catastrophic impact on the organization or individuals.

 • Low Sensitivity Data: Data intended for public use.



Methods of Data Classification

Data classification can be performed on the basis of content, context, or user selections:

 • Content-based Classification: It involves reviewing files and documents, and then subsequently classifying them.

 • Context-based Classification: It involves classifying files based on meta data like the application that created the file, the person who created the document, or the location in which files were modified.

 • User-based Classification: It involves classifying files according to a manual judgement of a knowledgeable user.



Regulatory Compliance

Compliance Regulations require organizations to protect specific data. Data classification enables you to identify the data subject to particular regulations so you can apply the required controls and pass audits. Here’s how data classification can help you meet common compliance standards:

 • GDPR — Data classification helps you uphold the rights of data subjects, including satisfying data subject access request by retrieving the set of documents with data about a given individual.

 • HIPAA — Knowing where all health records are stored helps you implement security controls for proper data protection.

 • ISO 27001 — Classifying information according to value and sensitivity helps you meet requirements for preventing unauthorized disclosure or modification.

 • NIST SP 800-53 — Categorizing data helps federal agencies properly architect and manage their IT systems.

 • PCI DSS — Data classification enables you to identify and secure consumer financial information used in payment card.

 • PDPA — The Personal Data Protection Act (PDPA) provides a baseline standard of protection for personal data. It complements sector-specific legislative and regulatory frameworks.
